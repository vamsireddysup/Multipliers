# Array multiplier (8-bit)

## Overview

An array multiplier computes the product of two unsigned binary numbers
by generating all partial products and summing them with an array of
adders. This is the most direct hardware implementation of the standard
long multiplication algorithm.

## Architecture

For two 8-bit operands a and b, each row i of the partial product array
represents `a` shifted left by i bit positions, but only contributes if
`b[i]` is 1:

```
pp[i] = b[i] ? (a << i) : 0,   for i = 0..7
```

The eight 16-bit partial product rows are then summed to produce the
final 16-bit product:

```
prod = pp[0] + pp[1] + ... + pp[7]
```

This is functionally identical to the original gate-level version, where
each `p[i][j] = a[j] & b[i]` term was generated by an individual AND gate
and the rows were summed by a ripple array of 56 full adders. The
addition here expresses the same row-by-row summation in a synthesizable
form without instantiating each adder explicitly.

## Critical path

Because the final step sums eight 16-bit rows, the critical path is
determined by the carry propagation through this addition, equivalent to
the carry ripple through the original full adder array. For an N-bit
operand, the worst case is roughly 2N-1 adder delays, which is why array
multipliers do not scale well to wider operands compared to tree-based
structures like Wallace or Dadda multipliers.

## Implementation

The implementation (array_multiplier_8bit.v) generates the eight partial
product rows with a generate loop, then sums them with a single addition
expression. The original structural version, built from an explicit 8x8
AND-gate grid and 56 full adder instances, is preserved in git history
for reference.
